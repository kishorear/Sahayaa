# Support AI Chat Widget Integration Manual

This manual provides detailed instructions for integrating the Support AI chat widget into your website, with complete information on authentication setup and AI configuration.

## Overview

The Support AI chat widget allows your website visitors to:

1. Log in with their personal accounts
2. Chat with your AI-powered support system
3. Receive personalized assistance based on their account information
4. Access the AI configuration you've set up in your Support AI dashboard

## Integration Steps

### Step 1: Extract the Files

1. Extract all files from the `client-integration` folder in the downloaded package
2. Upload these files to your web server or host them on a CDN

### Step 2: Include the Widget Script

Add the following code to your website's HTML, just before the closing `</body>` tag:

```html
<!-- Support AI Chat Widget Configuration -->
<script>
  window.supportAiConfig = {
    tenantId: YOUR_TENANT_ID,
    apiKey: "YOUR_API_KEY",
    requireAuth: true
  };
</script>

<!-- Support AI Chat Widget Script -->
<script src="path/to/auth-widget.js" async></script>
```

Replace the placeholders:
- `YOUR_TENANT_ID` - Your Support AI tenant ID (a number)
- `YOUR_API_KEY` - Your Support AI API key
- `path/to/auth-widget.js` - The actual path to where you uploaded the widget files

### Step 3: Customize the Widget (Optional)

You can customize the widget's appearance and behavior by adding these optional parameters to the configuration:

```javascript
window.supportAiConfig = {
  // Required parameters
  tenantId: YOUR_TENANT_ID,
  apiKey: "YOUR_API_KEY",
  
  // Optional parameters
  primaryColor: "6366F1",          // Color in hex format (without #)
  position: "right",               // Widget position: "right", "left" or "center"
  greetingMessage: "Hello!",       // Initial greeting message
  autoOpen: false,                 // Whether to auto-open on page load
  branding: true,                  // Show "Powered by Support AI" branding
  requireAuth: true,               // Require user authentication
  serverUrl: "https://..."         // Custom server URL (if not using default)
};
```

### Step 4: Test the Implementation

1. Open your website in a browser
2. Verify that the chat widget button appears in the specified position
3. Click the button to open the chat widget
4. Test the authentication flow by:
   - Creating a new user account
   - Logging in with existing credentials
5. Test the chat functionality:
   - Send test messages
   - Verify that responses are generated by your configured AI provider

## Authentication Flow

The widget provides a built-in authentication system that allows users to:

1. Register new accounts specific to your tenant
2. Log in with existing credentials
3. Maintain persistent login state across page visits
4. Securely communicate with your Support AI instance

All users who register through the widget are automatically associated with your tenant in the Support AI system. You can manage these users through your Support AI admin dashboard.

## API Integration

### Required API Endpoints

For the widget to function properly, ensure your Support AI instance supports these endpoints:

1. **User Authentication**:
   - `POST /api/widget/auth/login` - Handle user login
   - `POST /api/widget/auth/register` - Handle user registration

2. **Chat Processing**:
   - `POST /api/widget/chat` - Process messages and generate AI responses

3. **Analytics**:
   - `POST /api/widget/analytics` - Collect usage analytics

The standard Support AI installation already includes these endpoints, so no additional setup is needed if you're using the default configuration.

### Custom Server URL

If you're hosting Support AI on a custom domain, specify the `serverUrl` parameter in your configuration:

```javascript
window.supportAiConfig = {
  // ... other parameters
  serverUrl: "https://your-support-ai-instance.com"
};
```

## AI Provider Configuration

The widget automatically uses the AI provider configured in your Support AI admin dashboard. To configure or update your AI provider:

1. Log in to your Support AI admin dashboard
2. Navigate to "AI Configuration" settings
3. Select your preferred AI provider
4. Configure any provider-specific settings
5. Save your changes

The widget will automatically use your selected AI provider without requiring any additional code changes.

## Advanced Integration

### Programmatic Control

You can control the widget programmatically using the JavaScript API:

```javascript
// Initialize with custom configuration
window.SupportAIChat.init({
  tenantId: YOUR_TENANT_ID,
  apiKey: "YOUR_API_KEY",
  requireAuth: true
});

// Open the chat window
window.SupportAIChat.open();

// Close the chat window
window.SupportAIChat.close();

// Log out the current user
window.SupportAIChat.logout();
```

### Single Page Applications (SPAs)

The widget is compatible with SPAs built with React, Angular, Vue, or other frameworks. It uses a combination of techniques to maintain state across page transitions:

1. MutationObserver to detect DOM changes
2. History API integration for navigation events
3. URL monitoring for client-side routing changes

No special configuration is needed for SPA integration.

## Security Considerations

To ensure secure integration:

1. **HTTPS** - Host the widget on a secure (HTTPS) domain to protect user data
2. **API Key** - Keep your API key secure and rotate it periodically
3. **Cross-Origin Requests** - Configure CORS headers if hosting on a different domain
4. **Content Security Policy** - Update your CSP headers to allow the widget scripts

## Troubleshooting

### Widget Not Appearing
- Verify that the script is properly included in your HTML
- Check browser console for any JavaScript errors
- Ensure your API key and tenant ID are correct

### Authentication Issues
- Confirm that your Support AI instance is running correctly
- Check network requests for API call failures
- Verify that your server supports the required authentication endpoints

### Chat Not Working
- Check your AI provider configuration in the Support AI dashboard
- Ensure your server is correctly processing chat messages
- Look for CORS errors if hosting across different domains
